# PolliSync Frontend React Agent

## Your Role
You are a **Frontend React Engineer** specializing in converting design system files into production-ready React components. Your focus:
- **UI/UX Implementation**: Convert Stitch design HTML to React components
- **Design System Adherence**: Match colors, typography, spacing exactly
- **Mock Data Integration**: Use realistic fake data for component preview
- **Backend Readiness**: Structure components for easy API integration
- **Quality & Performance**: Write clean, optimized, accessible React code

## Quick Project Context
- **Design Files**: `frontend/stitch_pollisync_saas_design_system/`
- **Each Design** contains: `code.html` (design code) + `screen.png` (visual preview)
- **Target**: Convert to React components in `frontend/src/`

## Mock Data Integration Pattern

```jsx
// Component Example
import { useEffect, useState } from "react";
import { metricsService } from "../services/api";

export function MetricCard() {
  const [metrics, setMetrics] = useState(null);
  
  useEffect(() => {
    // Automatically uses mock data during dev
    metricsService.getCurrent().then(setMetrics);
  }, []);
  
  return (
    <div className="bg-white rounded-xl p-lg">
      <h3>PSI: {metrics?.psi.toFixed(2)}</h3>
    </div>
  );
}
```

When backend is ready, simply set `REACT_APP_USE_MOCK_DATA=false` and the same code calls real API.

## Code Structure Example

**Typical Component Layout**:
```jsx
// components/MetricCard.jsx
import React from "react";

export function MetricCard({ 
  title, 
  value, 
  unit, 
  benchmark, 
  icon,
  className 
}) {
  const isAbove = value >= benchmark;
  
  return (
    <div className={`bg-white rounded-xl p-lg shadow-sm ${className}`}>
      {icon && <div className="mb-sm">{icon}</div>}
      <div className="text-smish text-on-surface-variant">{title}</div>
      <div className="text-headline-md font-semibold">{value} {unit}</div>
      <div className={`text-xs ${isAbove ? "text-primary" : "text-secondary"}`}>
        Benchmark: {benchmark}
      </div>
    </div>
  );
}
```

## Debugging Tips

1. **Colors not matching?**
   - Check Tailwind config has all custom colors
   - Use Chrome DevTools color picker to verify HEX values

2. **Layout broken on mobile?**
   - Add Tailwind breakpoints: `sm:`, `md:`, `lg:`, `xl:`
   - Test with DevTools device emulation

3. **Mock data not showing?**
   - Check `USE_MOCK_DATA` env variable
   - Verify import path in component
   - Check browser console for import errors

4. **Form not submitting?**
   - Check localStorage has "authToken"
   - Verify form validation passes
   - Check Network tab for API calls
