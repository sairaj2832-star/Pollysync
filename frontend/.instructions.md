# PolliSync Frontend React Agent Instructions

## Overview
You are a frontend engineer responsible for converting the Stitch design system into production-ready React components. The design files are located in `frontend/stitch_pollisync_saas_design_system/*/` with corresponding `code.html` (design code) and `screen.png` (visual preview) for each screen.

Your mission:
- **Implement exact visual designs** from the Stitch files
- **Build reusable React components** following the design system
- **Use fake data for preview** that matches backend API structure
- **Prepare for backend integration** with clear data layer separation
- **Maintain responsive design** and accessibility standards

## Implementation Guidelines

### 1. **Component Structure**
```
src/
├── components/          # Reusable components
│   ├── common/         # Buttons, inputs, cards
│   ├── forms/          # Form components
│   ├── charts/         # Chart components
│   ├── dashboard/      # Dashboard-specific components
│   └── ui/             # UI primitives
├── pages/              # Page components (existing)
├── hooks/              # Custom React hooks
├── context/            # React Context (existing)
├── lib/                # Utilities and helpers
├── services/           # API call preparation
└── data/               # Fake data & mock functions
```

### 2. **Styling Approach**
- **Use Tailwind CSS** with the design system config (colors, spacing, fonts already defined)
- **Update `tailwind.config.js`** with the complete theme from design files if not already done
- **Create component-scoped CSS** using Tailwind classes
- **Ensure dark mode support** (design has dark mode variants)
- **No inline styles** - use Tailwind utility classes exclusively

### 3. **Fake Data Strategy**
- Create `src/data/mockData.js` with all fake datasets
- Export functions like:
  - `getMockUser()`
  - `getMockPredictions(count)`
  - `getMockMetrics()`
  - `getMockChartData()`
  - `getMockHistoryData()`
- Use `@faker-js/faker` lib for realistic data generation where needed
- **Data toggle**: Add a `USE_MOCK_DATA` environment variable (default: true for development)

### 4. **Backend Integration Points**
- Create `src/services/api.js` with placeholder functions:
  - `auth.login(email, password)`
  - `auth.register(userData)`
  - `auth.logout()`
  - `predictions.create(parameters)`
  - `predictions.list(filters)`
  - `predictions.getById(id)`
  - `metrics.getCurrent()`
  - `metrics.getHistorical(timeRange)`
- Functions should accept `USE_MOCK_DATA` flag to switch between mock and real data
- Return data in consistent format ready for component consumption

### 5. **State Management**
- Use **React Context** for global state (auth, user, theme)
- Use **useState** for component-level state
- Use **useReducer** for complex prediction workflows
- Prepare for Redux migration if needed

### 6. **Performance Considerations**
- Use `React.memo()` for expensive components
- Implement `useMemo` for computed values
- Use `useCallback` for event handlers passed to memoized children
- Lazy load dashboard charts with Suspense
- Implement pagination/virtualization for long lists

### 7. **Accessibility (a11y)**
- Semantic HTML (use proper heading hierarchy)
- ARIA labels for complex components
- Keyboard navigation support
- Color contrast compliance (WCAG AA minimum)
- Form labels associated with inputs

### 8. **Error Handling**
- Error boundary components for card failures
- User-friendly error messages
- Retry mechanisms for failed predictions
- Form validation with inline error messages

## Code Example: Button Component

```jsx
// components/common/Button.jsx
export function Button({ 
  variant = "primary", 
  size = "md", 
  children, 
  className,
  disabled,
  loading,
  ...props 
}) {
  const baseStyles = "rounded-lg font-medium transition-colors";
  
  const variants = {
    primary: "bg-primary text-white hover:bg-green-700 disabled:bg-gray-400",
    secondary: "bg-secondary text-white hover:bg-orange-700 disabled:bg-gray-400",
    outline: "border-2 border-outline text-on-surface hover:bg-surface-variant",
    error: "bg-error text-white hover:bg-red-700"
  };
  
  const sizes = {
    sm: "px-md py-xs text-sm",
    md: "px-lg py-md text-body-md",
    lg: "px-xl py-lg text-body-lg"
  };
  
  return (
    <button
      className={`${baseStyles} ${variants[variant]} ${sizes[size]} ${className}`}
      disabled={disabled || loading}
      {...props}
    >
      {loading ? "Loading..." : children}
    </button>
  );
}
```
